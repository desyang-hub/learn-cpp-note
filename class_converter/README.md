## c++ 类型转换

### 1.static_cast —— “老实巴交的搬运工”
> 一句话总结：只要逻辑通顺，且不需要运行时检查，就用它

### 2. dynamic_cast —— “小心翼翼的侦探”
> 一句话总结：父子转换怕出事，用它最安全（虽然慢点）。
常用于将基类指针或引用转换为派生类指针或引用，如果转换不成功则返回NULL（指针）或抛出异常（引用）。

### 3. const_cast —— “专门撕标签的”
> 一句话总结：除了加减 const 属性，它啥也不干。

### 4. reinterpret_cast —— “无视规则的暴力狂”
> 一句话总结：除非你非常清楚自己在干什么（比如操作硬件），否则别碰它。

### 总结
| 转换类型 | 核心用途 | 安全性 | 检查时机 | 形象比喻 |
| :--- | :--- | :--- | :--- | :--- |
| `static_cast` | 常规转换（数字、父子向上） | ✅ 安全 | 编译时 | 老实搬运工 |
| `dynamic_cast` | 安全的向下转型（多态） | ✅ 安全 | 运行时 | 谨慎侦探 |
| `const_cast` | 修改 const 属性 | ⚠️ 小心用 | 编译时 | 撕标签专员 |
| `reinterpret_cast`| 底层指针/整数互转 | ❌ 极危险 | 无检查 | 暴力狂 |
