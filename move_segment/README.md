## 移动语义


### 面试常问

1. 左值和右值的区别？什么是将亡值（xvalue）？
左值是指有变量名的值，右值是临时值，没有变量名。将亡值是即将被销毁的左值。

2. 移动构造和拷贝构造的区别？什么时候会自动生成移动操作？
移动构造作用的目标是一个右值引用，或是被标记为将亡值的左值引用。移动构造函数的参数类型是右值引用，而拷贝构造函数的参数类型是左值引用。当对象被移动时，会自动生成移动构造函数。


3. 为什么移动后要把原对象置为可析构状态？
移动语义的本意是将资源所有权从一个对象转移到另一个对象，因此需要将原对象置为可析构状态，以避免重复析构。

### 作用原理

移动构造/赋值函数，直接窃取源对象的资源，然后将源对象资源置为空状态，从而避免深拷贝带来的性能开销；

当使用std::move作用于一个对象时，该对象会被标记为将亡值，此时编译器会优先调用移动构造函数或移动赋值函数，而不是拷贝构造函数或拷贝赋值函数，之后该对象不能再被使用。


1. std::move做了什么？为什么它自己不移动任何东西？
std::move只是将一个对象标记为将亡值，它本身并不移动任何东西。
因为移动操作由移动构造或移动赋值函数来完成，而不是std::move本身。

2. 如何判断一个类是否该实现移动语义？
拷贝代价高，或不可用(无拷贝构造函数)
移动比拷贝显著提升性能场景


### std::move 和 std::forward
std::move是无条件将对象标记为右值引用
std::forward是有条件将对象标记为右值引用，仅当对象为右值引用时，才会将其标记为右值引用。用于完美转发。
