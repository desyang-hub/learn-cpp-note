## select poll epoll

select、poll、epoll都是linux中用于多路复用的组件，可以使得单线程下，同时处理多个IO事件(多个文件描述符的就绪状态)。

* select 是最早的实现，有文件描述符数量限制，最大为1024。
* poll 改进了select的局限性，但在大规模连接下，性能仍然不理想
* epoll 是Linux的特有实现，适用事件驱动模型，释放大量并发连接，性能优越

### select

- 适用位图来表示文件描述符集合，最大为1024
- 每次调用需要重新设置监控集合
- 采用轮询的方式检查文件描述符的就绪状态，时间复杂度是O(n)

### poll

- 改进select文件描述符数量限制
- 适用pollfd结构数组代替位图，没有最大文件描述符限制
- 仍然采用轮询的方式检查文件描述符的就绪状态，时间复杂度是O(n)

### epoll

- Linux特有高新能IO多路复用机制，基于事件驱动模型，无需轮询
- 采用红黑树来管理文件描述符，高效管理大量连接
- 就序列表返回已就绪文件描述符，时间复杂度为O(1)


### 总结

* select 是最原始的IO多路复用实现，使用位图存储文件描述符，最大为1024，采用轮询方式检查就绪状态
* poll 从select中改进了文件描述符数量限制，使用pollfd结构数组存储文件描述符，仍采用轮询方式检查就绪状态
* epoll 是Linux特有的高性能IO多路复用实现，基于事件驱动模型，使用红黑树管理文件描述符，无需轮询，时间复杂度为O(1)
